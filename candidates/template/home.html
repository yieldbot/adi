Introduction
 
spirit makes creating complex applications easy.
As an extension on top of datomic, spirit was especially designed to model data, relationships and their complexities.
 
So what is spirit?
 
   - Document and graph hybrid database.
   - Conversion of relational data to maps and vice-versa
   - Data-centric, declarative CRUD
   - Schema driven validation and processing.
   - Intuitive, simple, expressive.
 
The key to spirit lies the power of the schema. Instead of limiting the programmer, the schema should exhance him/her, much like what a type system does for programmers but without being as restrictive. spirit allows for relationships in data to be expressed as schema and then provides methods for smarter, more declarative control over data access.
Usage
 
In your project.clj add the following to include the library:
 
[im.chit/spirit "<@=version>"]
 
All functionality can be found in spirit.core:
 
(use 'spirit.core)
 
More information about packages can be found in the api section and the guides.
